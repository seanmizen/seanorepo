# SEED PROMPT — ffmpeg wrapper site

> One-file brief. Hand this to a coding agent (or future-me in a fresh session) and they should be able to build the thing without asking obvious questions. Opinionated on purpose. Where it says "DECIDE", make a call and stick to it.

---

## 1. The thesis (don't lose this)

Build the **agent-native ffmpeg wrapper**. Three doors into one engine:

1. **Humans via search** — programmatic SEO across every (input × output × operation) combo. Each page is a working tool, not an article.
2. **AI agents via MCP** — when someone asks ChatGPT/Claude to convert a video, our MCP server is the tool it calls.
3. **Devs via API** — metered REST API, every web action shows the equivalent ffmpeg command.

**Wedge:** privacy-first (wasm in-browser where possible), no watermark, no signup wall, no email gate. The hard wall is *volume*, not *quality*. Repeat visits + word-of-mouth are the SEO engine; we cannot afford to annoy anyone out of their second visit.

**Anti-thesis (what we are NOT):** Cloudconvert clone. We don't compete on "list of every format". We compete on *speed*, *agent reach*, and *not being annoying*.

## 4. Stack (recommended — change only with reason)

- **Frontend:** Next.js 15 (App Router), React, TypeScript. Static generation for all pSEO pages.
- **Styling:** Tailwind. No component library — design is part of the moat.
- **In-browser engine:** `@ffmpeg/ffmpeg` (ffmpeg.wasm). Use the **LGPL build** to stay clean on codec licensing. Avoid shipping H.264 encode in the wasm bundle if licensing is uncertain — server-side it.
- **Server engine:** ffmpeg in a Docker container, sandboxed. One container per job. Hard CPU/memory/time limits. Output to S3-compatible storage with TTL.
- **Backend:** Node or Go. Go if you expect the API to scale; Node if you want one language across the stack. DECIDE.
- **Queue:** Redis + BullMQ (Node) or Asynq (Go). Don't reinvent.
- **DB:** Postgres. Users, API keys, usage, jobs. That's it.
- **Hosting:** Vercel for the frontend (cheap pSEO scaling). Fly.io / Hetzner for the worker fleet (cheap CPU-heavy jobs).
- **Auth:** Clerk or Auth.js. Email + GitHub. Pro tier only — free tier is anonymous.
- **Payments:** Stripe. Subscriptions for Pro, metered billing for API.
- **Analytics:** Plausible (privacy-first, matches the brand). Server logs for ops metrics.

## 6. The operations matrix (the data model)

This is the seed for pSEO. One row per (operation, input, output, params). Generator builds N pages from M rows.

```ts
type Operation =
  | 'convert' | 'compress' | 'extract-audio' | 'extract-frames'
  | 'trim' | 'resize' | 'rotate' | 'gif' | 'merge' | 'mute'
  | 'change-speed' | 'add-subtitles' | 'remove-audio' | 'reverse';

type Format =
  | 'mp4' | 'mov' | 'webm' | 'mkv' | 'avi' | 'flv' | 'wmv' | 'm4v' | 'mpeg'
  | 'mp3' | 'wav' | 'flac' | 'aac' | 'ogg' | 'm4a' | 'opus'
  | 'gif' | 'webp' | 'apng';

type ToolPage = {
  slug: string;                  // 'mov-to-mp4'
  operation: Operation;
  inputFormats: Format[];        // ['mov']
  outputFormat: Format;          // 'mp4'
  title: string;                 // 'Convert MOV to MP4 — free, no watermark'
  h1: string;                    // 'MOV to MP4'
  intentVolume: 'head' | 'mid' | 'tail';
  ffmpegCommand: string;         // 'ffmpeg -i input.mov -c:v libx264 -c:a aac output.mp4'
  faqs: { q: string; a: string }[];
  related: string[];             // sibling slugs for internal linking
  preset?: { crf?: number; preset?: string; targetSizeMb?: number; resolution?: string; };
};
```

**Coverage target for v1:** at least 200 pages. Convert (15 video × 14 video) + extract-audio (15 video × 6 audio) + compress (15 video) + gif (15 → gif) + the size-targeted compress variants for mp4/mov/webm.

**Generation rule:** if a combo is technically nonsensical (e.g. mp3-to-mov), do not generate. Do not 404 — redirect to the closest valid page.

## 7. Page templates

2. **One sentence headline.** Example: *"Convert, compress, and trim video. In your browser. No watermarks."*

## 8. Performance budget (hard limits)

- **LCP:** < 1.2s on 4G. Tool pages especially.
- **JS shipped on tool page (initial):** < 80kb. ffmpeg.wasm loads on user interaction, not on page load.
- **CLS:** < 0.05.
- **TTFB:** < 200ms for static pSEO pages (CDN).
- **Lighthouse Performance:** ≥ 95 on every pSEO page.

If a feature breaks the budget, ship the feature behind interaction (button click) or don't ship it.

## 11. API spec (v1)

Minimum viable. Expand after launch.

```
POST /v1/jobs
  multipart: file
  json: { op: "convert", target: "mp4", options?: {...} }
  → 202 { job_id, status_url }

GET /v1/jobs/:id
  → 200 { status: "queued"|"running"|"done"|"error",
          download_url?, ffmpeg_command, error? }

GET /v1/usage
  → 200 { ops_this_month, ops_limit, plan }
```

Auth: `Authorization: Bearer <key>`. Keys created in dashboard.
Errors: standard JSON `{ error: { code, message } }`. Use HTTP codes properly.
Idempotency: accept `Idempotency-Key` header.

## 13. Things that will go wrong (anticipate)

- **Codec licensing.** ffmpeg.wasm + H.264/AAC has edges. Use LGPL build, document what we ship, get a lawyer's eyes on it before Pro launches.
- **Abuse.** People will try to convert TBs for free. Rate limit by IP + fingerprint, not just session. Cloudflare in front.
- **Cloudconvert / Convertio outranking us on head terms.** Expected. Win on long tail + speed + agent reach in year one. Branded search is the year-two goal.
- **Wasm performance on low-end Android.** Test on a £100 phone. Have a graceful "switch to server" option.
- **MCP directory churn.** The MCP ecosystem is young. Track where people actually discover MCPs and adjust.
